🧾 Генератор комерційних пропозицій СЕС + УЗЕ
Проблема

Менеджери з продажу витрачали 1–3 години на підготовку кожного комерційного КП вручну: зібрати параметри об'єкту, вибрати обладнання, порахувати потужність і економіку, заповнити шаблон Word, сконвертувати в PDF, надіслати клієнту. При 10–15 запитах на тиждень — це критичне навантаження, яке гальмувало швидкість відповіді і впливало на конверсію.

Збір вимог
1
Інтерв'ю з менеджерами і керівником продажів
4 режими розрахунку (за споживанням, потужністю, площею даху, вручну). Два напрямки: СЕС (сонячні станції від 100 кВт) і УЗЕ (установки збереження енергії від 233 кВт·год).
2
Опис бізнес-логіки
Greedy-алгоритм підбору інверторів. DC/AC ratio 1.28, ліміти 1.1–1.5. Кредитна логіка за диференційованою схемою (не ануїтет). 24 регіони → 5 зон сонячної інсоляції.
Розробка — два рішення
A
Universal Generator KP — веб-додаток
9-кроковий wizard для менеджерів зі смартфона. Каталог обладнання (6 панелей, 7 інверторів) у n8n static data. Панель адміна для оновлення каталогу без коду.
B
Odoo-модуль rayton_sale_kp
Вкладка "Генератор КП" прямо у картці договору в Odoo. Odoo → n8n webhook → Google Doc шаблон → batch заміна 50+ змінних → pdf.co pipeline → PDF назад в Odoo.
C
n8n воркфлоу — 90+ вузлів
SES гілка: Calculate → Charts (QuickChart API) → Копіювання шаблону → Docs batchUpdate → Видалення порожніх рядків → Вставка зображень → pdf.co → Telegram + Drive. Аналогічно для УЗЕ.

💬 Project Hub — Централізована комунікація
Проблема

Комунікація по кожному проекту була розпорошена: хтось пише в Telegram, хтось в Odoo Discuss, хтось в особисті. Керівник не бачив повну картину. Нові учасники підключалися вручну. При старті проекту — 5–10 хвилин на організацію групи і запрошення людей.

Сценарій роботи
1
Менеджер ініціює проект в Odoo
Заповнює форму: клієнт, тип проекту (СЕС / УЗЕ), координати, регіон, побажання клієнта. Один клік — "Ініціювати проект".
2
Odoo → n8n webhook → Telegram
Webhook передає параметри проекту. n8n через Telethon API створює нову Telegram-групу з назвою проекту та автоматично додає всіх учасників за їх TG username.
3
Перше повідомлення — структурований бриф
Бот надсилає в групу повний бриф проекту: тип, клієнт, координати, регіон, побажання. Всі учасники одразу мають контекст без додаткових пояснень.
4
Прив'язка Discuss-каналу до TG групи
Odoo Discuss-канал проекту прив'язується до Telegram-групи. Всі повідомлення з Discuss синхронізуються в TG і навпаки — команда може писати де зручно.
5
Завдання по етапах в Odoo Project
Автоматично створюються задачі по стандартних етапах: Замір → Фінрозрахунок → КП → Договір → Оплата → Доставка → Монтаж → Пуско-налагодження. Виконавці, дедлайни, прогрес — все в Odoo.
Технічна реалізація

Модуль rayton_project_hub на Python/Odoo 17. Кастомна форма ініціації з полями специфічними для бізнесу Rayton. Webhook-виклик до n8n при підтвердженні. n8n використовує Telethon (MTProto API) для створення груп — оскільки звичайний Bot API не дозволяє додавати учасників у групи без їх згоди. Callback-кнопки в боті для швидкої навігації між проектом в Odoo і TG групою.

Проблема

Менеджери дзвонили з особистих мобільних — жодної фіксації в CRM, жодних записів розмов, неможливо оцінити якість роботи КЦ. КЦ і відділ продажів — повністю "чорна скринька" для керівника.

Оцінка та перехід
1
Аналіз платформ і вибір Ringostat
Оцінено webhook-подієву модель, вартість, зберігання записів, API для інтеграцій. Ringostat обрано за нативну підтримку FMC і якість транскрипції.
2
Комунікація з Lifecell — FMC-номери
Корпоративні SIM-картки прив'язані до SIP-лінії АТС Ringostat. Менеджери зберегли звичні мобільні номери — всі дзвінки проходять через систему і записуються.
n8n воркфлоу — обробка дзвінків
→
Фільтрація внутрішніх номерів
83 виключені SIP/мобільні номери співробітників у Google Sheets — дзвінки між своїми не потрапляють у CRM.
→
Розподіл по відділах і типах
Switch-вузли: тип дзвінка (NO ANSWER / ANSWERED / BUSY) × відділ (Оператор КЦ / Менеджер / Кредитний). Для кожної комбінації — свій сценарій дії в CRM.
→
Знайдений / незнайдений контакт
Відомий номер → прив'язка до угоди. Невідомий → автоматичне створення контакту + нового ліда у КЦ з позначкою "потребує кваліфікації".
