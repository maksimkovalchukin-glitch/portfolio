Автоматизація · n8n · Google Docs API
🧾 Генератор комерційних пропозицій СЕС + УЗЕ
Проблема

Менеджери з продажу витрачали 1–3 години на підготовку кожного комерційного КП вручну: зібрати параметри об'єкту, вибрати обладнання, порахувати потужність і економіку, заповнити шаблон Word, сконвертувати в PDF, надіслати клієнту. При 10–15 запитах на тиждень — це критичне навантаження, яке гальмувало швидкість відповіді і впливало на конверсію.

Збір вимог
1
Інтерв'ю з менеджерами і керівником продажів
4 режими розрахунку (за споживанням, потужністю, площею даху, вручну). Два напрямки: СЕС (сонячні станції від 100 кВт) і УЗЕ (установки збереження енергії від 233 кВт·год). Фінал — PDF у чаттері угоди без ручних кроків.
2
Опис бізнес-логіки
Greedy-алгоритм підбору інверторів (заповнювати ≥100 кВт спочатку, потім малі). DC/AC ratio 1.28, ліміти 1.1–1.5. Кредитна логіка за диференційованою схемою (не ануїтет). 24 регіони → 5 зон сонячної інсоляції.
Розробка — два рішення
A
Universal Generator KP — веб-додаток
9-кроковий wizard для менеджерів зі смартфона. Каталог обладнання (6 панелей, 7 інверторів) у n8n static data. Панель адміна для оновлення каталогу без коду. Розрахунок кредиту, графіки окупності, завантаження логотипу клієнта.
B
Odoo-модуль rayton_sale_kp
Вкладка "Генератор КП" прямо у картці договору в Odoo. Odoo → n8n webhook → Google Doc шаблон → batch заміна 50+ змінних → видалення порожніх рядків → вставка графіків та зображень → pdf.co pipeline → PDF назад в Odoo.
C
n8n воркфлоу — 90+ вузлів
SES гілка: Calculate → Charts (QuickChart API, 1200×570px) → Копіювання шаблону → Docs batchUpdate → Видалення порожніх рядків → Вставка зображень → pdf.co (Upload → Delete pages → Add Logo → Binary) → Telegram підтвердження + Google Drive збереження. Аналогічно для УЗЕ.
Скріни
📸 n8n воркфлоу — повна схема пайплайну генерації (SES + UZE гілки)
📸 Telegram Mini App — вибір СЕС / УЗЕ
📸 Mini App — дашборд зі збереженими КП
📸 Odoo — вкладка Генератор КП у картці угоди
📸 Telegram Mini App — УЗЕ форма параметрів
Навчання та релізи

Записано навчальні відео для менеджерів по обох варіантах. Після першого релізу зібрано правки: real-time DC/AC ratio hint, виправлення помилки в ключах каталогу, калібровка pdf.co pipeline (~9 кредитів/запит). Повторний реліз задеплоєно через n8n API.

📞 Ringostat — IP-телефонія + AI-аналіз дзвінків
Проблема

Менеджери дзвонили з особистих мобільних — жодної фіксації в CRM, жодних записів розмов, неможливо оцінити якість роботи КЦ. КЦ і відділ продажів — повністю "чорна скринька" для керівника.

Оцінка та перехід
1
Аналіз платформ і вибір Ringostat
Оцінено webhook-подієву модель, вартість, зберігання записів, API для інтеграцій. Ringostat обрано за нативну підтримку FMC і якість транскрипції.
2
Комунікація з Lifecell — FMC-номери
Переговори щодо підключення Fixed Mobile Convergence: корпоративні SIM-картки прив'язані до SIP-лінії АТС Ringostat. Менеджери зберегли звичні мобільні номери — всі дзвінки проходять через систему і записуються.
n8n воркфлоу — Вхідні та Вихідні дзвінки

Кожен дзвінок → webhook у n8n → автоматична обробка за сценарієм:

→
Фільтрація внутрішніх номерів
83 виключені SIP/мобільні номери співробітників у Google Sheets — дзвінки між своїми не потрапляють у CRM.
→
Розподіл по відділах і типах
Switch-вузли: тип дзвінка (NO ANSWER / ANSWERED / BUSY) × відділ (Оператор КЦ / Менеджер / Кредитний). Для кожної комбінації — свій сценарій дії в CRM.
→
Знайдений / незнайдений контакт
Відомий номер → прив'язка до угоди → активність у чаттері. Невідомий номер → автоматичне створення контакту + нового ліда у КЦ з позначкою "потребує кваліфікації".
AI "Коротка вижимка розмови"

Після завершення дзвінка GPT-4 отримує транскрипцію (utterances) і генерує HTML-нотатку для CRM: суть розмови, потреби клієнта, запропоноване рішення, наступні кроки, ступінь зацікавленості клієнта у % (шкала 1–100%). Нотатка автоматично оновлює активність у Pipedrive/Odoo.

AI Скоркарта оцінки дзвінків (КЛС)

Збір вимог до системи оцінки: інтерв'ю з керівником КЦ → опис критеріїв → реалізація через 5 паралельних GPT-4 агентів зі Structured Output Parser (JSON):

Агент Блок оцінки Макс. балів
Greating Привітання + Завершення дзвінка 15
Needs Виявлення потреб (профіль ≥100 кВт / ≥233 кВт·год) 25
Solution Презентація рішення + Робота із запереченнями 35
Agreement Домовленість + Додатково 15
Summary Фінальна HTML-нотатка з total_score + зацікавленість % —

Розумна логіка: якщо клієнт нецільовий (перенаправляє, некомпетентний) — агент виставляє максимум по своєму блоку, щоб оператор не отримував штраф за ситуацію поза його контролем.

Проблема

Pipedrive покривав лише воронку продажів і не мав інструментів для кастомної логіки КЦ: розподіл лідів між операторами, черга менеджерів, передача між відділами, облік дзвінків, КПІ-аналітика. Кожна спроба зробити це у Pipedrive вимагала обхідних рішень.

Реалізовано
1
Ролі та права доступу (4 групи)
Оператор КЦ, Менеджер з продажу, Кредитний спеціаліст, Керівник КЦ. Кожна роль бачить тільки свою частину CRM.
2
Черга менеджерів і передача лідів
3 типи черги (KCC / incoming / credit), пауза менеджера, лічильник лідів. Wizard передачі ліда між операторами і менеджерами з логом усіх переміщень.
3
КПІ менеджерів — 15+ метрик
Дашборд по кожному менеджеру: ліди з комунікацією, заплановані активності, прострочені, зустрічі, дзвінки Ringostat (total / answered / missed / minutes).
4
Telegram ↔ Odoo двостороння синхронізація
Нативно через Telegram Bot API (без n8n). Текст, фото, документи, голосові — в обидва боки. 58 маппінгів TG username → Odoo user завантажені з Excel.
5
Міграція Pipedrive → Odoo (~90%)
Перенесення угод, контактів, компаній, активностей. Скрипт через Pipedrive API + Odoo RPC. ~1110 orphan-контактів у черзі + ЄДРПОУ збагачення.
